# Security Policy

## Supported Versions

We provide security updates for the following versions of AIOS:

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |
| < 0.1   | :x:                |

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, please report them via one of the following methods:

### Preferred Method: Email

Send an email to **contact@cuilabs.io** with subject "[SECURITY]" and include:
- A detailed description of the vulnerability
- Steps to reproduce the issue
- Potential impact assessment
- Suggested fix (if available)

### Alternative: GitHub Security Advisory

Use GitHub's [Security Advisory](https://github.com/cuilabs/aios/security/advisories/new) feature to report vulnerabilities privately.

## What to Expect

1. **Acknowledgment**: You will receive an acknowledgment within 48 hours
2. **Initial Assessment**: We will assess the vulnerability within 7 days
3. **Updates**: We will provide regular updates on the status of the fix
4. **Resolution**: We will notify you when the vulnerability is fixed

## Disclosure Policy

- We will work with you to coordinate disclosure
- We will credit you for the discovery (if desired)
- We will not disclose your identity without your permission
- We aim to fix critical vulnerabilities within 30 days

## Security Best Practices

When using AIOS:

1. **Keep Updated**: Always use the latest stable version
2. **Secure Configuration**: Follow security configuration guidelines
3. **Capability Tokens**: Use capability tokens properly
4. **Network Security**: Secure network communications
5. **Audit Logs**: Monitor audit logs regularly

## Security Features

AIOS includes the following security features:

- Post-quantum cryptography
- Capability-based security model
- Agent isolation and sandboxing
- Immutable audit logs
- Secure boot and attestation
- Behavioral anomaly detection (Enterprise Edition)

## Known Security Limitations

- AIOS is in early development (v0.1.x)
- Some security features are not yet fully implemented
- See [MISSING_COMPONENTS.md](MISSING_COMPONENTS.md) for details

## Security Updates

Security updates will be:
- Released as patch versions (e.g., 0.1.1, 0.1.2)
- Documented in [CHANGELOG.md](CHANGELOG.md)
- Announced via GitHub releases
